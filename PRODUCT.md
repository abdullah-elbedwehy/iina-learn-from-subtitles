# Product

## Register

brand

## Users

GitHub visitors, IINA users, and English learners who watch movies with subtitles. The primary user is a non-native English speaker who wants quick vocabulary help without leaving the movie flow for long.

## Product Purpose

Subtitle Lookup is a small IINA plugin that turns the current subtitle line into an AI vocabulary lookup. The website should help visitors understand the plugin quickly, trust that it is simple, and install or inspect the project from GitHub.

## Brand Personality

Clear, calm, practical. The tone should feel like a friendly macOS utility: precise enough for technical users, personal enough for learners.

## Anti-references

Avoid generic AI SaaS pages, oversized feature-card grids, noisy gradients, dark developer-tool aesthetics, and marketing copy that makes the plugin feel bigger than it is. Do not make the page feel disconnected from IINA.

## Design Principles

1. Explain the user moment before the feature list.
2. Keep the install path visible and low-friction.
3. Let the IINA relationship shape the visual language without copying the IINA site.
4. Show one concrete subtitle example so the value is obvious.
5. Keep the plugin feeling small, useful, and trustworthy.

## Accessibility & Inclusion

Target accessible contrast, semantic HTML, keyboard-friendly links and controls, responsive layout, and reduced-motion-friendly animation. Keep language simple for non-native English readers.
